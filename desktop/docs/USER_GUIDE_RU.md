# Trade Clipper — инструкция пользователя

## Что делает приложение

Trade Clipper автоматически сохраняет видео сделки: берет момент входа и выхода, добавляет несколько секунд до/после сделки и создает локальный MP4-файл с понятным названием.

Пример названия:

```text
2026-05-13_03-49-21_BINANCE_FUTURES_BTCUSDT_LONG.mp4
```

Формат имени:

```text
ГОД-МЕСЯЦ-ДЕНЬ_ЧАС-МИНУТА-СЕКУНДА_БИРЖА_ТИП-РЫНКА_МОНЕТА_НАПРАВЛЕНИЕ.mp4
```

Рядом позже будет создаваться JSON-метадата с параметрами сделки и путями.

## Текущий режим MVP

Сейчас заложен локальный безопасный конвейер:

1. Сделка закрыта.
2. Приложение знает время входа и выхода.
3. OBS Replay Buffer сохраняет последние минуты записи.
4. ffmpeg обрезает replay по окну сделки.
5. Клип попадает в локальную папку.
6. YouTube-загрузка остается ручной до подключения OAuth и финальной проверки.

Сейчас реализованы:

- русский интерфейс;
- тестовая сделка BTCUSDT для проверки пайплайна;
- расчёт окна клипа;
- генерация имён файлов и папок;
- планировщик ffmpeg-команды;
- базовые настройки приложения;
- OBS WebSocket boundary: подключение, проверка Replay Buffer и команда SaveReplayBuffer через IPC;
- сохранение базовых настроек в `settings.json`;
- быстрый экран настроек OBS host/port, папки клипов и отступов;
- пользовательская документация.

## Как пользоваться сейчас

1. Откройте Trade Clipper.
2. В блоке `Быстрые настройки` проверьте OBS host, port, папку клипов и отступы до/после сделки.
3. Нажмите `Сохранить`.
4. Проверьте карточку `OBS Replay Buffer`.
5. Нажмите `Проверить систему`.
6. Если OBS еще не подключен, приложение покажет: `Сначала подключите OBS WebSocket и включите Replay Buffer`.
7. Если OBS подключен и Replay Buffer активен, приложение отправит в OBS команду `SaveReplayBuffer`.
8. Когда появится реальная сделка, клип попадёт в `Очередь проверки`.
9. Нажмите `Предпросмотр`, чтобы проверить видео.
10. Нажмите `Загрузить`, когда YouTube-интеграция будет подключена и вы готовы отправить клип.

## Что нужно подготовить в OBS

1. Откройте OBS.
2. Включите Replay Buffer.
3. Настройте длительность буфера минимум больше максимальной ожидаемой сделки. Для коротких сделок хватит 2–5 минут, для длинных лучше 10–30 минут.
4. Включите OBS WebSocket.
5. Используйте стандартные параметры:
   - host: `127.0.0.1`
   - port: `4455`
6. Введите пароль OBS WebSocket в `Быстрые настройки` и нажмите `Сохранить`. Пароль сохраняется в системный keychain, а не в `settings.json`.
7. Перед торговлей нажмите Start Replay Buffer в OBS.

Важно: если сделка длится дольше буфера OBS, начало сделки уже не попадёт в сохранённый replay. Буфер должен покрывать всю сделку плюс запас до и после.

## Где лежат клипы

По умолчанию приложение использует системную папку данных Electron и подпапку `clips`.

В интерфейсе путь показывается в блоке `Пайплайн клипа`.

Пример структуры:

```text
~/Library/Application Support/Trade Clipper/clips/2026-05-13/
```

## Как считается обрезка видео

Формула:

```text
начало клипа = время входа - запас до сделки
конец клипа = время выхода + запас после сделки
```

Текущие значения по умолчанию:

- запас до входа: 3 секунды;
- запас после выхода: 5 секунд;
- Replay Buffer: 1800 секунд.

## Как будут подключаться сделки

Поддерживаются два варианта:

1. Через дневник сделок: Trade Clipper получает факт открытия/закрытия сделки из дневника.
2. Напрямую из биржи: Trade Clipper получает сделки через API биржи.

В обоих вариантах приложению нужны только времена входа/выхода, монета, биржа, рынок и направление сделки.

Для полной автоматизации можно предоставить:

- API-ключ дневника Trader Make Money или документацию по API;
- API-ключи бирж только с правами чтения истории сделок;
- данные OBS WebSocket;
- YouTube OAuth для загрузки видео;
- правило публикации: оставлять на проверке, загружать private, unlisted или public.

## Подписка и доступ

В продукт заранее заложена модель продажи по подписке:

1. Пользователь регистрируется через Telegram-бота.
2. Telegram-бот выдает лицензию или активирует подписку.
3. Промокод может давать доступ или скидку.
4. Промокод/подписка может требовать привязку Discord.
5. Сервер проверяет, состоит ли пользователь в нужном Discord-сервере.
6. Desktop-приложение получает только статус доступа, а не секреты платежной системы.

Подробный план: `desktop/docs/SUBSCRIPTION_TELEGRAM_DISCORD_PLAN_RU.md`.

## Безопасность

- Renderer не получает API-ключи напрямую.
- Привилегированные операции идут через Electron main process и узкий IPC API.
- OBS WebSocket пароль сохраняется в системный keychain через keytar и не пишется в JSON-настройки.
- Автоматическая загрузка в YouTube не включается, пока локальный пайплайн и права доступа не проверены.
- Биржевые ключи должны быть read-only, без разрешения торговли и вывода средств.

## Частые проблемы

### OBS не подключён

Проверьте, что OBS запущен, WebSocket включён, порт 4455 доступен, Replay Buffer стартовал.

### Клип не содержит начало сделки

Увеличьте длительность Replay Buffer в OBS. Буфер должен покрывать всю сделку плюс запас до и после.

### Название монеты или биржи выглядит странно

Имя файла специально очищается от небезопасных символов, чтобы файл корректно работал на macOS и Windows.

### YouTube не загружает клип

На текущем этапе YouTube работает как очередь ручной проверки. OAuth и загрузка подключаются отдельным шагом после локального пайплайна.
