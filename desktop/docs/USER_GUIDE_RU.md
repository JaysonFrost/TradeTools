# TradeTools: инструкция пользователя

## Что делает приложение

TradeTools автоматически сохраняет видео сделки: берет момент входа и выхода, добавляет несколько секунд до/после сделки и создает локальный MP4-файл с понятным названием.

Пример названия:

```text
2026-05-13_03-49-21_BINANCE_FUTURES_BTCUSDT_LONG.mp4
```

Формат имени:

```text
ГОД-МЕСЯЦ-ДЕНЬ_ЧАС-МИНУТА-СЕКУНДА_БИРЖА_ТИП-РЫНКА_МОНЕТА_НАПРАВЛЕНИЕ.mp4
```

Рядом с каждым `.mp4` создаётся `.json` файл с данными сделки, исходным replay-файлом и окном обрезки.

## Как работает запись

TradeTools работает локально и может использовать два источника видео:

- встроенная запись выбранного окна терминала;
- OBS Replay Buffer, если вам удобен OBS.

Общий конвейер:

1. Сделка закрыта.
2. Приложение знает время входа и выхода.
3. Источник записи отдаёт replay последних секунд/минут.
4. Встроенный ffmpeg обрезает replay по окну сделки.
5. Клип попадает в локальную папку.
6. Клип остается в локальной очереди до удаления из очереди или ручного использования файла.

Сейчас реализованы:

- русский интерфейс;
- тестовая сделка BTCUSDT для проверки пайплайна;
- расчёт окна клипа;
- генерация имён файлов и папок;
- планировщик ffmpeg-команды;
- базовые настройки приложения;
- встроенная запись окна терминала без OBS;
- OBS WebSocket boundary: подключение, проверка Replay Buffer и команда SaveReplayBuffer через IPC;
- сохранение базовых настроек в `settings.json`;
- быстрый экран настроек источника записи, папки клипов и отступов;
- отдельный мастер `Мастер настройки видео` для источника записи, папок, источника сделок и тестового клипа;
- отдельный мастер `Мастер настройки прокси` для двух серверов, связки, SSH-проверки и запуска локального proxy;
- реальный локальный pipeline: replay из выбранного источника -> ffmpeg trim -> видео + JSON metadata;
- очередь клипов, которая читает созданные metadata JSON;
- локальное хранилище торговых прокси с датами оплаты и ссылками на личные кабинеты;
- автозапуск при входе в систему;
- системные уведомления об оплате прокси и успешной записи сделки;
- пользовательская документация.

## Как пользоваться сейчас

1. Откройте TradeTools.
2. В левой панели откройте `Видео`.
3. В верхней панели нажмите `Мастер настройки видео`.
4. Пройдите экраны мастера:
   - быстрый старт;
   - источник записи;
   - проверка буфера;
   - папки и отступы;
   - источник сделок;
   - тестовый клип;
   - готово.
5. На шаге `Источник записи` выберите `Встроенная запись окна`, выберите окно терминала и нажмите `Сохранить этот шаг`.
6. Если хотите использовать OBS, выберите `OBS Replay Buffer`, впишите host, port и пароль OBS, затем нажмите `Сохранить этот шаг`.
7. На шаге проверки нажмите `Проверить видео`.
8. На шаге `Папки и отступы` укажите папку клипов, запас до входа и после выхода, затем нажмите `Сохранить этот шаг`. Для OBS дополнительно укажите `Папка OBS replay`.
9. На шаге `Создайте тестовый клип` нажмите `Создать тестовый клип`, чтобы приложение взяло replay, обрезало его через ffmpeg и записало metadata JSON.
10. Если источник записи не готов, приложение покажет, что нужно открыть окно терминала или запустить OBS Replay Buffer.
11. Если всё настроено, новый клип появится в очереди клипов.
12. Нажмите `Предпросмотр`, чтобы проверить видео.
13. Нажмите `Открыть файл`, чтобы перейти к готовому клипу в папке клипов.

## Режим без OBS

1. Откройте торговый терминал.
2. В TradeTools откройте `Видео` → `Запись видео`.
3. Выберите `Встроенная запись окна`.
4. Нажмите `Обновить` рядом со списком окон.
5. Выберите окно терминала.
6. Оставьте FPS 30, если нет причины менять.
7. Нажмите `Сохранить`.

Окно терминала должно оставаться открытым. Если окно закрыть или переименовать так, что TradeTools его не найдёт, запись остановится до повторного выбора окна.

На macOS система может запросить разрешение записи экрана для TradeTools.

## Если нужен OBS

1. Откройте OBS.
2. Включите Replay Buffer.
3. Настройте длительность буфера минимум больше максимальной ожидаемой сделки. Для коротких сделок хватит 2-5 минут, для длинных лучше 10-30 минут.
4. Включите OBS WebSocket.
5. Используйте стандартные параметры:
   - host: `127.0.0.1`
   - port: `4455`
6. Введите пароль OBS WebSocket в `Быстрые настройки` и нажмите `Сохранить`. Пароль сохраняется в системный keychain, а не в `settings.json`.
7. В OBS откройте настройки вывода/записи и запомните папку, куда сохраняется Replay Buffer.
8. Введите эту папку в поле `Папка OBS replay` в TradeTools.
9. Перед торговлей нажмите Start Replay Buffer в OBS.

Важно: если сделка длится дольше буфера OBS, начало сделки уже не попадёт в сохранённый replay. Буфер должен покрывать всю сделку плюс запас до и после.

## Где лежат клипы

По умолчанию приложение использует системную папку данных Electron и подпапку `clips`.

В интерфейсе путь показывается в настройках видео и в карточке готового клипа.

Пример структуры:

```text
~/Library/Application Support/TradeTools/clips/2026-05-13/
```

## Как считается обрезка видео

Формула:

```text
начало клипа = время входа - запас до сделки
конец клипа = время выхода + запас после сделки
```

Текущие значения по умолчанию:

- запас до входа: 2 секунды;
- запас после выхода: 2 секунды;
- буфер записи: 1800 секунд.

## Как будут подключаться сделки

Основной источник сделок для автоматизации — read-only API биржи.

Сейчас в приложении есть Binance USDT-M Futures: TradeTools получает открытые позиции и историю закрытых сделок через API, а затем создаёт локальный клип.

Приложению нужны только времена входа/выхода, монета, биржа, рынок и направление сделки.

Для полной автоматизации можно предоставить:

- API-ключи бирж только с правами чтения истории сделок;
- настроенный источник записи: встроенное окно или OBS;
- правило обработки: оставлять клип в очереди проверки или использовать файл вручную.

## Прокси-серверы и связки

В левой панели откройте `Прокси`. В разделе `Прокси-серверы` пользователь добавляет свои серверы:

1. название;
2. IP или домен;
3. SSH-логин;
4. SSH-пароль;
5. ссылку на сайт хостинга;
6. число оплаты в месяце.

Пароли серверов сохраняются только в системный keychain и не пишутся в `settings.json`.

Чтобы собрать маршрут через мастер:

1. Нажмите `Мастер настройки прокси`.
2. На шаге добавления заполните два сервера: первый входной и второй выходной.
3. Нажмите `Сохранить два сервера и связку`.
4. Проверьте маршрут `первый сервер -> второй сервер`.
5. Нажмите `Проверить SSH`, если нужно только проверить доступы.
6. Нажмите `Настроить и запустить связку`, если нужно автоматически установить серверную связку и поднять локальный proxy для терминала.

Если серверов больше двух, добавьте их на странице `Прокси` и переставьте в блоке `Порядок связки`.

Приложение подключится по SSH к каждому серверу из маршрута, проверит сохранённые доступы и покажет инструкцию:

- что добавить в торговый терминал после автоматической настройки: тип `HTTP`, host `127.0.0.1`, локальный порт, логин и пароль пустые.

Автоматическая настройка ставит Xray на каждый VPS, пишет VLESS TCP конфигурацию на `443/tcp`, связывает каждый сервер со следующим и оставляет последний сервер выходным. На компьютере TradeTools скачивает/запускает Xray core и поднимает локальный HTTP proxy `127.0.0.1:1083` или порт, выбранный в первом сервере связки. Перед проверкой и настройкой TradeTools также смотрит VPN/туннели, системный proxy и маршрут к первому VPS. После настройки TradeTools покажет локальные host/port для торгового терминала, диагностику маршрута и проверки exit IP/Binance.

Дополнительный proxy-клиент для автоматической схемы не нужен: торговый терминал подключается к локальному HTTP proxy, который запускает TradeTools.

Если пинг остаётся высоким, проверьте блок `VPN и маршрут` после проверки/настройки. При найденном VPN или антизапрете нажмите `Обойти VPN для VPS` на странице `Прокси`. На Windows TradeTools запросит UAC и добавит persistent `/32` маршруты до VPS через обычный Wi-Fi/Ethernet gateway, чтобы локальный Xray подключался к серверам напрямую, а не через TUN/VPN. После этого перезапустите связку и торговый терминал.

Если VPN-клиент поддерживает split tunneling, всё равно полезно исключить TradeTools и локальный Xray core из туннеля. На macOS автоматическое добавление маршрутов пока не выполняется: добавьте host routes до VPS вручную или используйте split tunneling в VPN-клиенте.

Для пошагового сценария нажмите `Мастер настройки прокси` в верхней панели страницы `Прокси`.

## Безопасность

- Renderer не получает API-ключи напрямую.
- Привилегированные операции идут через Electron main process и узкий IPC API.
- OBS WebSocket пароль, если используется OBS-режим, сохраняется в системный keychain и не пишется в JSON-настройки.
- Пароли прокси сохраняются в системный keychain и не пишутся в `settings.json`.
- Биржевые ключи должны быть read-only, без разрешения торговли и вывода средств.

## Частые проблемы

### Встроенная запись окна не стартует

Проверьте, что окно терминала открыто и выбрано в `Видео` -> `Запись видео`. На macOS проверьте разрешение записи экрана для TradeTools.

### OBS не подключён

Эта проблема относится только к OBS-режиму. Проверьте, что OBS запущен, WebSocket включён, порт 4455 доступен, Replay Buffer стартовал.

### Клип не содержит начало сделки

Увеличьте длительность буфера записи. Для OBS это Replay Buffer, для встроенной записи это поле `Replay Buffer` в настройках клипа. Буфер должен покрывать всю сделку плюс запас до и после.

### Название монеты или биржи выглядит странно

Имя файла специально очищается от небезопасных символов, чтобы файл корректно работал на macOS и Windows.
