# Автоматический VPN bypass и упрощённый экран прокси

## Цель

TradeTools остаётся единственной точкой настройки для торговых терминалов: терминал использует HTTP proxy `127.0.0.1:<порт TradeTools>`, а локальный Xray отправляет трафик в торговую цепочку. При активном Happ, v2RayTun или другом TUN-клиенте соединение локального Xray с торговым VPS должно идти напрямую через обычный Wi-Fi/Ethernet, не меняя маршрутизацию остального VPN-трафика.

Экран прокси должен показывать ежедневный сценарий подключения без потока технических логов.

## Границы

Входит в работу:

- Windows: автоматическая проверка bypass при старте локального proxy и после изменения сети или TUN-состояния.
- Точные IPv4 host routes только к адресам, найденным в активной локальной Xray-конфигурации.
- Happ и v2RayTun через общую проверку интерфейсов и маршрутов, без интеграции с их внутренними API.
- Упрощение страницы прокси и разделение действий «подключить» и «проверить».

Не входит:

- Изменение настроек торговых терминалов.
- Изменение default route, DNS, системного proxy или правил VPN.
- Служба Windows, драйвер, планировщик задач либо постоянный привилегированный процесс.
- Автоматическое изменение чужих `/32` маршрутов.
- Автоматическое добавление маршрутов на macOS/Linux.

## Подходы

### Рекомендуемый: периодическая сверка во время работы proxy

Когда сохранённый локальный Xray активен, TradeTools сверяет сетевое состояние при старте и затем с коротким интервалом. Новый fingerprint запускает сверку маршрутов. UAC появляется только если нужный маршрут отсутствует или устарел.

Плюсы: не зависит от API Happ/v2RayTun, переживает их смену, не требует установки службы и использует существующий механизм elevated PowerShell. Минус: реакция на смену сети занимает до одного интервала проверки.

### Событийный помощник Windows

Отдельный PowerShell/.NET-процесс слушает `NetworkAddressChanged` и вызывает сверку сразу.

Плюс: реакция без задержки. Минусы: ещё один долгоживущий процесс, сложнее жизненный цикл и обработка сбоев. Не выбираем: выигрыш не оправдывает дополнительную инфраструктуру.

### Привилегированная служба Windows

Один раз установить службу, которая применяет маршруты без последующих UAC.

Плюс: полностью бесшумное обновление. Минусы: постоянные повышенные права, установочный сценарий и заметно больше поверхность безопасности. Не выбираем.

## Архитектура

### Источник VPS-адресов

Новый чистый разборщик читает `%APPDATA%/tradetools/xray-runtime/trade-chain.json` и извлекает IPv4-адреса или DNS-имена из VLESS outbound `vnext`. DNS-имена резолвятся в IPv4, дубликаты удаляются. Активный файл Xray — источник истины: в текущей архитектуре он содержит именно первый VPS, к которому подключается компьютер; дальнейшие hop-соединения происходят уже между VPS и не проходят через пользовательский VPN.

Если конфигурация отсутствует, невалидна или адрес не резолвится, TradeTools не меняет маршруты и отображает понятный диагностический статус.

### Снимок сети

Модуль обхода создаёт fingerprint из:

- активных адаптеров и обнаруженных TUN-интерфейсов;
- обычного IPv4 default gateway и его interface index;
- текущего маршрута до каждого целевого VPS;
- списка целевых IP.

TUN определяется текущим общим шаблоном адаптеров, включая `tun`, `wintun`, `v2ray` и другие известные варианты. Поэтому Happ и v2RayTun поддерживаются одинаково, без хрупкой привязки к имени процесса.

Пока локальный proxy работает, монитор проверяет fingerprint каждые 15 секунд. Одинаковый fingerprint ничего не делает. При изменении запускается одна последовательная сверка; параллельные запуски объединяются. Монитор останавливается вместе с локальным Xray/приложением.

### Сверка маршрутов

1. Выбирается активный default route через не-TUN Wi-Fi/Ethernet адаптер с IPv4 gateway.
2. Для каждого IP проверяется `/32` маршрут.
3. Если маршрут уже ведёт через выбранные gateway и interface, действие не требуется.
4. Если маршрут отсутствует, создаётся persistent `/32` маршрут с метрикой 1.
5. Если маршрут ранее был создан TradeTools и gateway/interface изменились, он заменяется на новый.
6. Если маршрут принадлежит не TradeTools, он не меняется; в диагностике показывается причина.

TradeTools хранит минимальный реестр собственных маршрутов (адрес, gateway, interface index, время). Реестр нужен только для безопасного отличия собственного устаревшего маршрута от чужого. Он не содержит секретов.

Сама запись маршрутов продолжает выполняться через временный elevated PowerShell. Если для нового или устаревшего маршрута нужны права, появляется UAC. При отмене UAC статус фиксируется для текущего fingerprint, чтобы не показывать одно и то же окно каждые 15 секунд; новая сеть или явное «Обновить» разрешают повторную попытку.

### Запуск

Порядок при старте приложения:

1. Восстановить сохранённый локальный Xray proxy.
2. После его успешного запуска начать монитор VPN bypass.
3. Немедленно выполнить первую сверку.

При ручном «Подключить прокси» Xray запускается/обновляется и затем та же сверка выполняется автоматически. Терминалу по-прежнему не нужен ни отдельный VPN, ни отдельная настройка после смены сети.

## Экран прокси

Страница строится вокруг двух блоков.

### Подключение

Верхняя карточка показывает только:

- состояние: «Подключено», «Подключаем», «Требуется действие» или «Не подключено»;
- маршрут цепочки;
- параметры терминала: `HTTP`, `127.0.0.1`, порт;
- краткое состояние VPN bypass: защищён, проверяется, ожидает UAC или требует внимания;
- время последней проверки.

Основная кнопка — **«Подключить прокси»**. Она безопасно повторяема: настраивает или обновляет серверную цепочку лишь при необходимости, запускает локальный Xray и затем сверяет bypass. Она не переустанавливает Xray на VPS, если действующая конфигурация совпадает.

Вторичная кнопка — **«Проверить подключение»**. Она выполняет только SSH/сетовые проверки и не меняет серверы, Xray или маршруты. Явная кнопка **«Обновить VPN bypass»** доступна в карточке статуса для повторной попытки после отменённого UAC.

Технические сообщения и построчный прогресс перемещаются в закрытый по умолчанию раздел «Диагностика». В основном сценарии отображается только текущий шаг и итог.

### Серверы

Список серверов, редактирование, порядок цепочки и операции управления располагаются в отдельном сворачиваемом разделе «Серверы цепочки». Это сохраняет текущие возможности, но не перегружает основной экран подключения.

## Ошибки и безопасность

- Нет активного физического gateway: маршруты не меняются, в статусе — просьба подключить Wi-Fi/Ethernet.
- Нет TUN/VPN: сверка показывает, что bypass не требуется; существующие собственные маршруты не удаляются автоматически.
- DNS/VLESS-config ошибка: маршруты не меняются.
- Чужой `/32` маршрут: не заменяется.
- Отмена UAC: не считается ошибкой Xray и не нарушает локальный proxy.
- Все привилегированные операции остаются в Electron main process; renderer получает только результат.

## Проверка

Добавить unit-тесты для:

- извлечения VPS из локальной Xray-конфигурации и дедупликации;
- определения необходимости создать, оставить, заменить собственный маршрут или оставить чужой;
- подавления повторного UAC для неизменного fingerprint;
- запуска/остановки монитора вместе с локальным Xray;
- разделения действий «Подключить прокси» и «Проверить подключение»;
- скрытия детального прогресса до раскрытия диагностики.

Ручная проверка на Windows: включить Happ и v2RayTun по очереди, сменить Wi-Fi/Ethernet или gateway, подтвердить UAC при необходимости и убедиться, что маршрут первого VPS использует физический интерфейс, а терминал продолжает работать через тот же `127.0.0.1:1083`.
